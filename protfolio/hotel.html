專案簡介

這個專案最初是為了模擬真實訂房網站的核心流程而做：訪客可以瀏覽不同房型、 選擇入住與退房日期、即時看到總價計算結果，並完成訂房送出，練習表單邏輯、 日期運算與資料驗證，做出符合實際使用情境的響應式介面。

完成第一版之後，我常常想像自己有一天也想開一間屬於自己的民宿，也希望能有更多人透過這樣的介面認識台灣、 喜歡上台灣。於是我把整個介面重新設計了一次：搜尋列改成「登機證」造型， 虛線分隔線與兩側圓形缺口，讓查房這件事多了一點出發旅行的儀式感； 房型卡片改成帶有郵戳徽章的「明信片」質感；最重要的房型詳情彈窗， 則改版成完整的登機證雙聯設計——左邊是房型故事，右邊票根放評價、可入住人數與房價， 並加上一欄「管家在地小提醒」，把台灣人情味具體地寫進每一個房型裡。 我也新增了「在地探索」區塊，用明信片的方式介紹九份、日月潭、太魯閣、阿里山， 附上一句在地管家的真心話，希望旅人在訂房的那一刻，就已經開始想念台灣。

功能亮點

01

登機證搜尋票卡 ＋ 明信片房型卡

首頁搜尋列改為登機證造型，用虛線分隔入住日期、退房日期與人數，兩側加上圓形缺口模擬票根撕開的細節；房型改以台灣風景命名（日月潭、九份、太魯閣、阿里山），卡片右上角貼上熱門／親子友善／尊爵臻選的郵戳徽章，呈現明信片般的溫度。

CSS Grid Responsive CSS 偽元素造型
02

日期選擇、總價計算與表單驗證

訪客選擇入住與退房日期後，系統自動計算住宿天數與總價並即時顯示；送出前檢查必填欄位與日期邏輯（退房日期需晚於入住日期），錯誤時提示訪客修正，確保資料正確才完成訂房。

JavaScript Date 即時運算 表單驗證
03

房型詳情彈窗：登機證雙聯設計

這次優化的重點。彈窗改成完整的登機證雙聯版面：左側是房型照片、故事化描述與設施標籤，右側票根呈現評價、可入住人數與房價，中間用虛線＋圓形缺口＋條碼裝飾做出撕票感；並新增「管家在地小提醒」欄位，讓旅人在做決定前，就先感受到一句貼心的在地提醒。

Modal 重構 CSS 缺口效果 UX 微文案
04

在地探索：融入台灣文化體驗

新增「在地探索」明信片牆，用九份、日月潭、太魯閣、阿里山四個角落，搭配一句由管家角色寫下的真心話，把「想讓更多人認識台灣」的初衷，實際變成介面上看得到、讀得到的內容，而不只是一句行銷標語。
