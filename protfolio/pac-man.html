PAC-MAN BAKERY 小精靈烘焙遊戲

從純 CSS 動畫練習，升級成具備遊戲邏輯與互動機制的網頁小遊戲。

✦ Overview

專案簡介

本專案最初的目標是練習 CSS Animation， 使用純 HTML 與 CSS 製作出小精靈沿著固定路徑移動、 自動吃掉麵包的動畫效果。

完成動畫版本後，進一步思考如何將作品從展示型動畫， 升級成真正能讓使用者操作的互動遊戲，因此重新設計整體架構， 導入 JavaScript 控制角色移動、碰撞偵測、 遊戲計分與時間管理等功能， 最終完成一款具備完整遊戲流程的 PAC-MAN BAKERY。

功能亮點

01

從動畫版升級為遊戲版

原始版本利用 CSS Keyframes 讓小精靈依照固定路徑移動， 麵包也會在指定時間自動消失。 後續移除自動播放機制， 改由 JavaScript 控制角色位置， 讓玩家能夠親自操作並完成遊戲。

CSS Animation JavaScript
02

玩家操作系統

支援鍵盤方向鍵、WASD 操作， 同時提供滑鼠點擊移動與手機版 Touch 控制， 讓遊戲在桌機與行動裝置都能順利遊玩。

Keyboard Event Touch Event Responsive
03

碰撞偵測與計分系統

透過 getBoundingClientRect() 實作碰撞偵測， 當小精靈碰到麵包時會自動吃掉並增加分數， 當所有食物被吃完後即獲得勝利。

Collision Detection Score System
04

倒數計時器

新增 60 秒挑戰模式， 玩家必須在時間結束前吃完所有麵包， 提升遊戲的緊張感與挑戰性。

setInterval() Game Timer
05

鬼魂追逐模式

加入 AI 鬼魂敵人， 會持續朝玩家方向移動。 若鬼魂碰觸到小精靈， 遊戲即結束， 增加遊戲性與挑戰難度。

Enemy AI Path Following
06

音效與重新開始機制

利用 Web Audio API 製作吃到麵包與勝利提示音， 並加入重新開始按鈕， 讓玩家能夠快速重新挑戰。

Web Audio API Restart System

✦ Learning

專案收穫

這次專案最大的收穫， 是學習如何將一個純動畫作品， 逐步拆解並重構成真正具有互動性的遊戲。

在開發過程中， 練習了事件監聽、 碰撞偵測、 計時器控制、 遊戲狀態管理、 響應式設計以及程式模組化思維， 也更加理解 JavaScript 在前端互動開發中的應用方式。
